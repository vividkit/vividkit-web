## Report Naming Convention

Format: `{role}-{YYMMDD}-{description}.{ext}`

- **Role:**
  - `tester` - Comprehensive testing reports
  - `debugger` - Focused debugging reports
  - `docs-manager` - Documentation and phase reports
  - `lighthouse` - Performance audit data

- **Date:** YYMMDD format (e.g., 251206 = December 6, 2025)

- **Description:** Descriptive title in kebab-case

## Maintenance

- New reports added with each significant change
- Previous reports preserved for historical reference
- Reports are version controlled with the codebase
- Summaries created for complex report data
